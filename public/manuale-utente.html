Panoramica del Sito

Il sito Sindrome ReNU Italia APS è la piattaforma ufficiale dell'associazione italiana per la Sindrome ReNU, condizione genetica ultra-rara causata da varianti patogene del gene RNU4-2, scoperta nell'ottobre 2024. È costruito su Cloudflare Pages con tecnologia Hono v4 (TypeScript) e database Cloudflare D1 (SQLite serverless con replica globale).

🌐

https://sindromerenu-italia.pages.dev
Il sito reindirizza automaticamente alla versione italiana: /it/home. Tutte le pagine sono disponibili in 5 lingue con URL strutturati come /{lingua}/{pagina}.

🧬 Cos'è ReNU — Informazioni sulla Sindrome

URL: /it/about

  • Barra di ricerca integrata nella hero section
  • Gene RNU4-2: RNA non codificante componente del macchinario di splicing, altamente conservato evolutivamente, localizzato in regione non codificante (non rilevabile con WES)
  • Dati epidemiologici: ~250 casi accertati a livello mondiale, 12-14 in Italia, prevalenza stimata ~1:35.000 nati vivi, varianti patogene concentrate in 13 posizioni del genoma
  • Data scoperta: ottobre 2024 (uno studio di Nature)
  • 8 categorie cliniche: anomalie cerebrali strutturali, disabilità intellettiva severa, epilessia/crisi convulsive, problemi visivi, caratteristiche dismorfiche del viso, ipotonia muscolare, problemi di mobilità, difficoltà di alimentazione/deglutizione
  • Avviso diagnostico critico: solo WGS (Whole Genome Sequencing) può rilevare le varianti RNU4-2; il WES standard non è sufficiente
  • Link al sito renusyndrome.org per approfondimenti internazionali

🩺 Diagnosi — WGS vs WES e Percorso Diagnostico

URL: /it/diagnosis

⚠️
Informazione critica per le famiglie

Il gene RNU4-2 si trova in una regione non codificante del genoma. Il WES (Whole Exome Sequencing) analizza solo il ~2% del genoma (regioni codificanti) e non rileva queste varianti. Richiedere sempre il WGS completo.

  • Tabella comparativa WGS vs WES: copertura, costo, sensibilità, interpretazione, disponibilità in Italia
  • Informazioni centri diagnostici italiani con invito a contattare info@sindromerenu.it
  • CTA pulsante rosso "Vuoi maggiori informazioni?" → pagina Contatti
  • Nota sulla valutazione prenatale: disponibile per varianti RNU4-2, yield diagnostico sotto il 40%

Flusso completo dopo l'invio del form Contatti
  • Il browser mostra uno spinner "Invio in corso..."
  • Il server valida tutti i campi (GDPR consent, nome ≥ 2 car., email valida, messaggio ≥ 10 car.)
  • I dati vengono salvati nella tabella contatti del database D1 con: timestamp UTC, IP hashato (SHA-256), versione privacy policy (v2.0), data consenso
  • Il server registra l'operazione nell'audit_log con azione "INSERT"
  • Viene inviata una email HTML a info@sindromerenu.it con tutti i dati e link reply
  • Viene inviata una email HTML di conferma all'utente con riepilogo e contatti
  • Il browser mostra messaggio verde "✅ Messaggio inviato!" e resetta il form
  • In caso di errore: messaggio rosso con link alternativo a info@sindromerenu.it

Validazione lato server (API Hono)

Tutta la validazione è implementata nell'API Hono (Cloudflare Workers). Non è possibile aggirare la validazione disabilitando JavaScript nel browser. In caso di errori il server risponde:

HTTP 201 — Successo

Dati salvati correttamente nel DB. Email inviate. Risposta JSON con success: true e messaggio di conferma.

HTTP 400 — Errore validazione

Risposta JSON con campo errors contenente i campi in errore e le motivazioni (es. "Consenso obbligatorio (Art. 6 GDPR)").

HTTP 500 — Errore server

Errore interno (es. DB non raggiungibile). Il messaggio invita a contattare info@sindromerenu.it.

HTTP 401 — Non autorizzato

Endpoint admin richiamato senza o con token non valido.

🚫
Protezione GDPR obbligatoria

Il campo consenso_gdpr è verificato server-side prima di qualsiasi inserimento nel database. Il constraint SQL consenso_gdpr INTEGER NOT NULL garantisce che non possano esistere record senza consenso esplicito, neanche in seguito a errori o manipolazioni dell'API.

Scheda Cancella Dati — Art. 17 GDPR (Diritto all'Oblio)

Permette di cancellare tutti i dati personali di un interessato in risposta a una richiesta formale.

⚠️
Operazione irreversibile — verificare l'identità prima di procedere

La cancellazione sovrascrive nome, cognome, email e messaggio con [CANCELLATO] e imposta il flag cancellato=1. L'ID e la data di creazione rimangono per garantire l'integrità dell'audit log. L'operazione è registrata nell'Audit Log con nota "Art.17 GDPR".

Procedura cancellazione dati (Art. 17)

  1. 1Ricevere la richiesta scritta dall'interessato (email o modulo fisso) con copia documento d'identità
  2. 2Verificare l'identità confrontando l'email con quella nel database
  3. 3Aprire il pannello admin e cliccare sulla scheda "Cancella Dati"
  4. 4Inserire l'indirizzo email dell'interessato nel campo apposito
  5. 5Digitare esattamente CANCELLA (maiuscolo) nel campo di conferma
  6. 6Cliccare "Esegui Cancellazione" — appare il riepilogo dei record cancellati
  7. 7Verificare il riepilogo — l'operazione è registrata nell'Audit Log con nota "Art.17 GDPR"
  8. 8Rispondere all'interessato entro i 30 giorni previsti dall'Art. 12.3 GDPR

Query SQL utili — Cloudflare D1 Console

Le seguenti query possono essere eseguite nella Console D1 di Cloudflare Dashboard:

-- Tutti i contatti ricevuti (più recenti prima)
SELECT id, created_at, nome, email, oggetto, status
FROM contatti WHERE cancellato=0
ORDER BY created_at DESC;

-- Lista attesa soci (non cancellati)
SELECT id, nome, cognome, email, citta, tipo, created_at
FROM lista_attesa WHERE cancellato=0
ORDER BY created_at DESC;

-- Ultime 20 operazioni audit (più recenti prima)
SELECT timestamp, tabella, azione, operatore, note
FROM audit_log
ORDER BY timestamp DESC LIMIT 20;

-- Statistiche: conta record per tabella
SELECT 'lista_attesa', COUNT(*) FROM lista_attesa WHERE cancellato=0
UNION ALL SELECT 'contatti', COUNT(*) FROM contatti WHERE cancellato=0
UNION ALL SELECT 'adesioni', COUNT(*) FROM adesioni WHERE cancellato=0
UNION ALL SELECT 'donazioni', COUNT(*) FROM donazioni;

-- Verifica consensi GDPR (tutti devono essere 1)
SELECT 'contatti', consenso_gdpr, COUNT(*)
FROM contatti GROUP BY consenso_gdpr;

GDPR e Privacy — Conformità Italiana

Il sito è progettato per essere conforme al Regolamento UE 2016/679 (GDPR) e al D.Lgs. 196/2003 (Codice Privacy italiano), come modificato dal D.Lgs. 101/2018. Di seguito le misure implementate e le procedure operative.

⚖️
Condivisibilità del database — Risposta alla domanda iniziale

Il database NON è condivisibile liberamente in quanto contiene dati personali (nomi, email, IP hashati). È accessibile solo tramite il pannello admin autenticato o la console Cloudflare (accesso riservato al Titolare e agli autorizzati). Questo è un requisito del GDPR (Art. 5 — principio di integrità e riservatezza).

Misure tecniche di sicurezza implementate

  • HTTPS obbligatorio: tutte le comunicazioni avvengono tramite TLS. Nessun dato viaggia in chiaro
  • Consenso GDPR obbligatorio: ogni form richiede la checkbox di consenso esplicito — validata sia frontend che backend — non è possibile salvare dati senza consenso (CHECK constraint SQL)
  • IP anonimizzato: l'indirizzo IP viene hashato con SHA-256 e salt "renu26:" prima del salvataggio — non è possibile risalire all'IP originale
  • Versione privacy policy tracciata: ogni consenso registra la versione della privacy policy accettata (testo_consenso_versione = '2.0')
  • Data e ora del consenso: salvata per ogni record con timestamp UTC
  • Audit log: ogni accesso e modifica ai dati personali viene registrato (Art. 5 GDPR — accountability)
  • Diritto all'oblio (Art. 17): cancellazione tramite admin panel con pseudonimizzazione completa ([CANCELLATO])
  • Pannello admin protetto da token: nessun dato accessibile senza autenticazione — HTTP 401
  • Dati minori (Art. 9): campo separato consenso_dati_minore nella tabella adesioni per dati del bambino con diagnosi
  • Cookie solo tecnici: nessun cookie di profilazione, tracking o analisi — solo cookie di sessione strettamente necessari
  • Dati fisicamente in EU: database Cloudflare D1 regione ENAM
